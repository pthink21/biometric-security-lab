🕵️ Tấn Công 1: Trinh Sát (Reconnaissance)

Cách Hoạt Động

  • Attacker gửi GET /users (không cần auth)
  • Server trả về JSON chứa danh sách tất cả user
  • Attacker lấy được tên nạn nhân

Code Attacker

import requests resp = requests.get("http://localhost:5000/users") users = resp.json()["users"] # Output: [{"name": "Nguyen Van A", ...}]

Vấn Đề Bảo Mật

❌ Information Disclosure - Endpoint không yêu cầu authentication

💉 Tấn Công 2: Injection Attack

Quy Trình

  • Bước 1: Lấy ảnh khuôn mặt nạn nhân (từ mạng)
  • Bước 2: Encode ảnh thành base64
  • Bước 3: POST trực tiếp tới /authenticate
  • Bước 4: Server xác thực thành công ❌

Payload

POST /authenticate HTTP/1.1 Content-Type: application/json { "image": "data:image/jpeg;base64,/9j/4AAQSkZJ..." }

Tại Sao Thành Công?

❌ Không có Liveness Detection - Server không kiểm tra ảnh có phải từ khuôn mặt sống thật không

🛡️ Khuyến Nghị Bảo Mật

1. Liveness Detection

  • Yêu cầu chuyển động (nháy mắt, cười, quay đầu)
  • Phát hiện ảnh 2D vs khuôn mặt sống (dùng AI/ML)

2. Authentication & Authorization

  • Endpoint /users cần JWT token (chỉ admin)
  • Endpoint /authenticate vẫn công khai nhưng kết hợp Liveness

3. Rate Limiting & Logging

  • Giới hạn request (5 lần/phút)
  • Ghi log tất cả xác thực + IP address
  • Phát hiện pattern tấn công

🎓 Kết Luận

  • ✅ Sinh trắc học khuôn mặt là tốt nhưng cần bảo vệ bổ sung
  • ✅ Liveness Detection là bước quan trọng để chống tấn công
  • ✅ Authentication trên tất cả endpoint là cần thiết
  • ✅ Bảo mật sinh trắc học = kết hợp nhiều lớp bảo vệ
